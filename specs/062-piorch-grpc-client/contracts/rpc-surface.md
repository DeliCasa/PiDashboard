# RPC Surface Contract: PiDashboard → PiOrchestrator

**Protocol**: Connect (JSON over HTTP/1.1 POST)
**Base Path**: `/rpc/`
**Package**: `delicasa.device.v1`

## Services & Methods

### CameraService

| RPC | URL Path | Auth | Request Fields | Response Fields |
|-----|----------|------|----------------|-----------------|
| ListCameras | POST `/rpc/delicasa.device.v1.CameraService/ListCameras` | None | correlationId | correlationId, cameras[], totalCount |
| GetCamera | POST `/rpc/delicasa.device.v1.CameraService/GetCamera` | None | correlationId, deviceId | correlationId, camera |
| GetCameraStatus | POST `/rpc/delicasa.device.v1.CameraService/GetCameraStatus` | None | correlationId, deviceId | correlationId, deviceId, status, lastSeen, ready |
| ReconcileCameras | POST `/rpc/delicasa.device.v1.CameraService/ReconcileCameras` | X-API-Key | correlationId | correlationId, reconciledCount, reconciledDeviceIds[] |

### CaptureService

| RPC | URL Path | Auth | Request Fields | Response Fields |

## Request Headers

| Header | Required | Source |
|--------|----------|--------|
| Content-Type | Always | `application/json` (set by Connect transport) |
| Connect-Protocol-Version | Always | Set automatically by `@connectrpc/connect-web` |
| X-API-Key | Protected RPCs only | `VITE_API_KEY` env var or `getApiKey()` |
| X-Correlation-ID | Optional | Set by interceptor; auto-generated by server if omitted |

## Error Codes

| Connect Code | HTTP Equivalent | PiDashboard Handling |
|-------------|-----------------|---------------------|
| not_found | 404 | Graceful degradation (feature unavailable) |
| unavailable | 503 | Service down — show retry |
| unauthenticated | 401 | Auth error — prompt for API key |
| permission_denied | 403 | Auth error — API key lacks permission |
| deadline_exceeded | 504 | Timeout — show retry |
| invalid_argument | 400 | Input validation error — show message |
| internal | 500 | Generic error — show retry |
